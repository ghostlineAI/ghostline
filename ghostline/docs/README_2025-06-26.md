# GhostLine Documentation

This repository contains comprehensive documentation for the GhostLine platform.

## Overview

This documentation covers:
- Project charter and vision
- Architecture Decision Records (ADRs)
- API documentation
- User guides and tutorials
- Developer documentation
- Operations runbooks
- Security and compliance docs

## Documentation Structure

```
docs/
├── PROJECT_CHARTER.md     # Project vision and goals
├── adr/                   # Architecture Decision Records
│   └── ADR-0000.md       # Foundational decisions
├── api-docs/             # API specifications
├── user-guides/          # End-user documentation
├── developer-guides/     # Developer documentation
├── runbooks/            # Operational procedures
└── security/            # Security documentation
```

## Key Documents

### Project Charter
Defines project vision, personas, KPIs, and success metrics.

### Architecture Decision Records
- **ADR-0000**: Foundational decisions (AWS, token billing, multi-agent)
- Future ADRs will document evolving architectural choices

## Contributing

When adding documentation:
1. Follow markdown best practices
2. Run markdown linter before committing
3. Keep documents concise and well-structured
4. Update table of contents when adding new sections

## Building Documentation Site

```bash
# Install dependencies
npm install

# Build documentation site
npm run build-docs

# Serve locally
npm run serve-docs
```

## Style Guide

- Use ATX-style headers (`#`, `##`, etc.)
- Keep lines under 120 characters
- Use tables for structured data
- Include diagrams where helpful
- Provide code examples
